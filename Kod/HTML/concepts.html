Begrepp

Den tekniska strukturen som utgör webben är givetvis otroligt komplicerad, men här görs ett försök att uttrycka det någorlunda enkelt.

    Terminologi om webben

  1. Hypertext, används som en sorts information som med hjälp av en dator möjliggör att flytta fokus från exempelvis en del av ett dokument till ett annat avsnitt i samma dokument. Det är även möjligt för ett helt annat dokument. Hyperlänkar, som fungerar som en slags referenser läggs in i dokumentet.
  2. Informationsidentiferare som är globalt standardiserade möjliggör att hitta en särskilt sorts information som finns lagrad på internet.
  3. Modellen mellan klient och server, som bidrar till att datorer kan samspela och kommunicera med varandra. Klientdatorns program kan utföra förfrågningar hos ett annat program på en serverdator. Serverdatorn ger därefter klienten den information som begärs.
  4. HTML och XHTML är också en viktig del av webben, och framför allt särskilt sådan när det gäller tillgänglighet för användaren. Denna text finns lagrad i ett dokument, men syns inte direkt för användaren på samma sätt som kodskaparen. I stället kan användaren få ta del av informationen genom att ett program tolkar dokumentet och presenterar innehållet smidigare och enklare. Det kan handla om kapitel och paragrafer, semantisk struktur och hur det ska visas grafiskt.

Standarder

Det finns tre generella standarder som utgör webben vi surfar på idag:

  • Uniform Resource Locator, URL styr hur webbaddresser ser ut.
  • Hyper Text Transfer Protocol HTTP styr hur webbsidorna skickas
  • Hyper Text Markup Language HTML som styr webbsidornas innehåll.

Vad som hör till webben är inte så definerat. En del gamla resurser som finns kan uppfattas tillhöra dessa men använder ej dessa standarder egentligen. World Wide Web Consortium är en organization som arbetar med att ta fram standarder för World Wide Web.
